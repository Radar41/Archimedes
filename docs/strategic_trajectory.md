# Strategic Trajectory

This document records the current sequence logic for the remaining strategic
work so the repo reflects the same gating used in Asana.

## Dependency Map

1. Use Archimedes/Hydra for real research work.
2. Capture friction data from that usage.
3. Turn the artifacts and friction data into a technical writeup.
4. Use the artifacts and writeup to support outreach, applications, and course planning.

This replaces the earlier assumption that outreach could run in parallel with
artifact production.

## Dual-Track North Stars

- Track A: produce artifacts through the governed pipeline.
- Track B: approach labs, employers, and other external stakeholders only after Track A yields evidence.

Track B is gated by Track A. Infrastructure alone is not the credential.

## Future-Use Architectural Relevance

Archimedes/Hydra may still serve as:

- a personal cognitive continuity system
- a human-centered orchestration substrate
- an epistemic governance layer for multi-agent workflows
- a testbed for memory, routing, evidence, and review patterns
- a source of empirical friction data about current AI interaction limits

It is not yet evidence of broad autonomy, scalability, or production generality.
